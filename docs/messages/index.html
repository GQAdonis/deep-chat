Messages

Properties related to messages.

initialMessages​

  • Type: MessageContent[]

Messages that are pre-populated when the chat loads up. This is useful for displaying previous messages and continuing a conversation.

Example​

  • Sample code
  • Full code
<deep-chat
  initialMessages='[
    {"text": "Show me a modern city", "role": "user"},
    {"file": {"src": "path-to-file.jpeg", "type": "image"}, "role": "ai"},
    {"text": "Whats on your mind?", "role": "user"},
    {"text": "Peace and tranquility", "role": "ai"}
  ]'
></deep-chat>
<!-- This example is for Vanilla JS and should be tailored to your framework (see Examples) -->

<deep-chat
  initialMessages='[
    {"text": "Show me a modern city", "role": "user"},
    {"file": {"src": "path-to-file.jpeg", "type": "image"}, "role": "ai"},
    {"text": "Whats on your mind?", "role": "user"},
    {"text": "Peace and tranquility", "role": "ai"}
  ]'
  containerStyle='{"height": "370px", "borderRadius": "8px"}'
  directConnection='{"demo": true}'
></deep-chat>

introMessage​

  • Type: string

Initial ai message that is displayed when the chat is loaded. It is different to initialMessages as it will not trigger the onNewMessage listener, will not be part of the returned messages when the getMessages method is called and is not taken into consideration when calculating requestBodyMessageLimits.

Example​

  • Sample code
  • Full code
<deep-chat introMessage="Hi I am your assistant, ask me anything!"></deep-chat>
<!-- This example is for Vanilla JS and should be tailored to your framework (see Examples) -->

<deep-chat
  introMessage="Hi I am your assistant, ask me anything!"
  directConnection='{"demo": true}'
  containerStyle='{"borderRadius": "8px"}'
></deep-chat>

avatars​

  • Type: true | {default?: Avatar, ai?: Avatar, user?: Avatar}

Add avatars beside messages. You can enable this by setting the value to true or by defining an object with Avatar properties. The default property is applied to both roles.

Example​

  • Sample code
  • Full code
<deep-chat avatars="true"></deep-chat>
<!-- This example is for Vanilla JS and should be tailored to your framework (see Examples) -->

<deep-chat
  avatars="true"
  initialMessages='[
    {"text": "Hey, how are you?", "role": "user"},
    {"text": "I am doing great, how about you?", "role": "ai"},
    {"text": "What is the meaning of life?", "role": "user"},
    {
      "text": "This ultimately depends on the person, but it could be the pursuit of happiness or fulfillment.",
      "role": "ai"
    }
  ]'
  containerStyle='{"borderRadius": "8px"}'
  directConnection='{"demo": true}'
></deep-chat>

Avatar​

  • Type: {
         src?: string,
         styles?: {
             avatar?: CustomStyle,
             container?: CustomStyle
             position?: "left" | "right"
    }}

This object is used to configure avatars that are displayed beside messages.
src is the path to an avatar image file.
avatar property is used to configure the avatar element's styling.
container is used to style the element that encapsulates the avatar element.
position defines which side of the message the avatar is going to be displayed.

Example​

  • Sample code
  • Full code
<deep-chat
  avatars='{
    "default": {"styles": {"avatar": {"height": "30px", "width": "30px"}, "container": {"marginTop": "8px"}}},
    "ai": {"src": "path-to-file.svg", "styles": {"avatar": {"marginLeft": "-3px"}}}
  }'
></deep-chat>

errorMessages​

  • Type: {
         displayServiceErrorMessages?: boolean,
         overrides: {
             default?: string,
             service?: string,
             speechToText?: string
    }}

  • Default: { displayServiceErrorMessages: false, overrides: {default: "Error, please try again." }}

Deep Chat automatically displays an error message when something goes wrong. This object is used to control what the message will contain.
displayServiceErrorMessages is used to display the full error message that has been returned from the service.
overrides is used to overwrite the error message text based on what has caused it: default is used for any kind of error, service is for communication with the target service and speechToText is when there is a problem using the speechToText. functionality.

// WORK - speechToText link

Example​

  • Sample code
  • Full code
<deep-chat
  errorMessages='{
    "overrides": {
      "default": "Default error :(",
      "service": "Target service error!",
      "speechToText": "Speech to text microphone error!"
    }
  }'
></deep-chat>

Types​

Shared object property types related to messages:

MessageContent​

  • Type:
         | {role: MessageRole, text: string}
         | {role: MessageRole, file: MessageFile}

Object format used to encapsulate each message within the chat. It uses EITHER ONE of the object types above where text or file property is defined.
The text string uses Markdown syntax to render the message accordingly. This is useful for displaying code snippets, paragraphs etc. See this playground for reference.

Example​

  • Sample code
  • Full code
<deep-chat
  initialMessages='[
    {
      "text": "```java\nwhile (i < 5) {\n console.log(\"hi\");\n i+= 1;\n}\n```",
      "role": "ai"
    }
  ]'
></deep-chat>
<!-- This example is for Vanilla JS and should be tailored to your framework (see Examples) -->

<deep-chat
  messageStyles='{
    "default": {
      "shared": {
        "bubble": {"maxWidth": "270px"}
      }
    }
  }'
  initialMessages='[
    {
      "text": "```java\nwhile (i < 5) {\n console.log(\"hi\");\n i+= 1;\n}\n```",
      "role": "ai"
    }
  ]'
  containerStyle='{"borderRadius": "8px"}'
  directConnection='{"demo": true}'
></deep-chat>

MessageRole​

  • Type: "ai" | "user"

Entity that the message is from.

MessageFile​

  • Type:
      | {name?: string, type?: MessageFileType, src: string}
      | {name?: string, type?: MessageFileType, base64: string}

  • Default: {name: "file", type: "any"}

Object format that defines the details for a single file.
The name property displays the file name inside the message bubble when type is not defined or is set to "any".
You can optionally define its type to help the component to render it inside an interative media bubble.
Interactive media bubbles also need EITHER src (path to the file) or base64 (binary encoding).

Example​

  • Sample code
  • Full code
<deep-chat
  initialMessages='[
    {"file": {"src": "path-to-file.jpeg", "type": "image"}, "role": "user"},
    {"file": {"src": "path-to-file.wav", "type": "audio"}, "role": "user"},
    {"file": {"src": "path-to-file", "name": "hello-world.txt", "type": "file"}, "role": "user"}
  ]'
></deep-chat>
<!-- This example is for Vanilla JS and should be tailored to your framework (see Examples) -->

<deep-chat
  initialMessages='[
    {"file": {"src": "path-to-file.jpeg", "type": "image"}, "role": "user"},
    {"file": {"src": "path-to-file.wav", "type": "audio"}, "role": "user"},
    {"file": {"src": "path-to-file", "name": "hello-world.txt", "type": "file"}, "role": "user"}
  ]'
  containerStyle='{"borderRadius": "8px"}'
  directConnection='{"demo": true}'
></deep-chat>
info

Files with "image" or "any" types that contain a src property will have their bubble set as a hyperlink.

MessageFileType​

  • Type: "image" | "audio" | "any"

Type of a file stored inside a message. "any" is the default type.
